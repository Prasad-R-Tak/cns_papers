Q1a
Explain stepwise the procedure carried out by DHCP client and DHCP server.
7 Marks
▾
✍ Answer
Definition
DHCP (Dynamic Host Configuration Protocol) is a network management protocol that automatically assigns IP addresses and other network configuration parameters to devices on a network. It operates over UDP using Port 67 (server) and Port 68 (client), eliminating the need for manual IP configuration.
DHCP DORA Process — Complete Stepwise Procedure

The entire process of IP address assignment is called the DORA process — Discover, Offer, Request, Acknowledge:

DORA FLOW DIAGRAM
CLIENT (Port 68) DHCP SERVER (Port 67) | | |─── Step 1: DHCPDISCOVER (broadcast) ──────►| | [src: 0.0.0.0 → dst: 255.255.255.255] | | | |◄── Step 2: DHCPOFFER ──────────────────────| | [Offered IP: 192.168.1.5, Lease: 24hr] | | | |─── Step 3: DHCPREQUEST (broadcast) ────────►| | [Accepting IP from chosen server] | | | |◄── Step 4: DHCPACK ────────────────────────| | [Confirmed! IP assigned successfully] | | | | IP ASSIGNED AND ACTIVE ✅ |
D
DHCPDISCOVER (Client → Broadcast): When a new device joins the network, it has no IP address. It sends a broadcast message to 255.255.255.255 asking "Is there any DHCP server available?" All devices receive this broadcast. The packet includes the client's MAC address for identification.
O
DHCPOFFER (Server → Client): The DHCP server receives the DISCOVER message. It selects an available IP from its IP address pool and replies with an OFFER containing: Offered IP address, Subnet mask, Default gateway, DNS server address, and Lease duration (e.g. 24 hours).
R
DHCPREQUEST (Client → Broadcast): The client receives the offer and sends a REQUEST message back confirming acceptance. This is still sent as a broadcast so that other DHCP servers know this client has chosen a particular server and their offers can be withdrawn.
A
DHCPACK (Server → Client): The server sends an ACKNOWLEDGEMENT confirming the IP assignment. The client now officially owns the IP for the lease duration. If the server cannot assign (IP taken), it sends DHCPNAK (Negative Acknowledgement).
IP Lease Renewal Process

T=0 T=50% T=87.5% T=100% | | | | IP ASSIGNED → Renewal → Any server IP Released attempt attempt back to pool (same server) (any DHCP)
What DHCP Assigns to Client

DHCP uses UDP (not TCP) because during DISCOVER, the client has NO IP address yet — it cannot establish a TCP connection. UDP broadcast is the only option!
Q1b
Define DNS. Explain how name resolution happens in DNS. Enlist all the resource records and its functions.
8 Marks
▾

Definition of DNS
DNS (Domain Name System) is a hierarchical, distributed, and decentralized naming system that translates human-readable domain names (e.g., www.google.com) into machine-readable IP addresses (e.g., 142.250.190.46). It uses Port 53, primarily UDP for queries and TCP for zone transfers or large responses.

Without DNS, users would need to memorize IP addresses for every website they visit.
DNS Hierarchical Structure
DNS HIERARCHY
ROOT ( . ) / | \ / | \ .com .org .edu .in .net .gov / \ \ google.com who.org mit.edu / \ www.google mail.google Level 1: Root DNS Level 2: TLD Level 3: Authoritative Level 4: Subdomains
How Name Resolution Happens — Step by Step

DNS resolution involves Recursive queries (client asks resolver to do all work) and Iterative queries (resolver asks each server one step at a time):

1
Local Cache Check: User types www.google.com. Browser checks its local DNS cache first. If found (TTL not expired), IP is returned immediately — no further queries needed.
2
Local DNS Resolver: If not cached, query goes to the Local DNS Resolver (assigned by ISP or DHCP). The resolver also checks its own cache. It acts on behalf of the client.
3
Root DNS Server: Resolver queries one of the 13 Root DNS servers. Root does NOT know the IP but responds: "Ask the .com TLD server at this address."
4
TLD DNS Server: Resolver queries the .com TLD server. It responds: "I don't know the IP, but ask Google's authoritative DNS at this address."
5
Authoritative DNS Server: Resolver queries Google's Authoritative DNS. This server KNOWS the answer and returns: IP = 142.250.190.46.
6
Response to Client: Resolver caches the result (for TTL duration) and returns the IP to the client's browser. Browser connects to 142.250.190.46 and loads the page ✅.
RESOLUTION FLOW
USER → Resolver → Root DNS → TLD DNS (.com) → Auth DNS → IP returned (checks ("ask .com ("ask google (gives exact cache) TLD server") auth server") IP address)
DNS Resource Records (RR) — All Types and Functions

Resource Records are data entries stored in DNS servers. Each RR has a Name, Type, TTL, and Value:

Record TypeFull NameFunctionExample

AAAAIPv6 Address RecordMaps domain name to IPv6 addressgoogle.com → 2607:f8b0::...

CNAMECanonical NameCreates an alias pointing to another domain namewww → google.com
NSName ServerIdentifies the authoritative DNS server for the domaingoogle.com NS = ns1.google.com

DNS is called "Distributed" because no single server stores all records — they are spread across millions of servers worldwide. It is "Hierarchical" because it follows Root → TLD → Authoritative structure.
DNS = Phonebook of Internet | Domain Name = Person's Name | IP = Phone Number. Root servers = 13 sets labeled A to M.

Why FTP Requires Two TCP Ports

FTP uses two separate TCP connections because it separates control information from actual data transfer, providing better efficiency and flexibility:

FTP TWO-CONNECTION MODEL
CLIENT ═══ Control Connection (Port 21) ═══ SERVER ← Commands: USER, PASS, RETR, QUIT (stays open entire session) CLIENT ─── Data Connection (Port 20) ─── SERVER ← Actual file content (opens per transfer, closes after)
  • Port 21 (Control): Stays open for the entire FTP session. Used for all text-based commands and server responses. Allows commands to be sent even while a file is transferring.
  • Port 20 (Data): Opens only when a file transfer is needed, closes immediately after each transfer. Separation allows abort commands to be sent via control channel while data is flowing.

Limitations of Bluetooth
  • Short Range: Maximum range is only about 10 meters (Class 2), severely limiting its use to personal area networks only.
  • Low Data Rate: Maximum speed is only about 3 Mbps (Classic Bluetooth), which is insufficient for high-bandwidth applications like video streaming.
  • Limited Number of Devices: A piconet supports only 1 master + 7 active slaves. Only 8 devices can be active simultaneously in one piconet.
  • Interference: Operates in the 2.4 GHz ISM band — same as WiFi and microwave ovens — causing potential interference and performance degradation.
  • Security Concerns: Vulnerable to attacks like Bluejacking (sending unsolicited messages), Bluesnarfing (unauthorized access to data), and Bluebugging (taking control of device).
  • Master-Slave Architecture: All communication must go through the master device. Slaves cannot communicate directly with each other, creating a bottleneck.
  • Pairing Complexity: Initial pairing process can be complicated for non-technical users. Devices need to be in discoverable mode and within close range.
  • Not Suitable for Internet: Cannot directly provide internet connectivity unlike WiFi. Primarily designed for peripheral connectivity.
802.11 = WLAN (building/home), 802.15 = WPAN (personal bubble). Bluetooth's biggest limitations are its short range and low speed compared to WiFi!

MAC Sub-Layers Overview
The IEEE 802.11 MAC layer has two sub-layers for controlling channel access: DCF (Distributed Coordination Function) — mandatory, and PCF (Point Coordination Function) — optional.
1. DCF — Distributed Coordination Function

DCF is the primary and mandatory access method in 802.11. It uses CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance):

DCF TIMING
───[DIFS]──[Backoff]──[DATA]──[SIFS]──[ACK]────► (wait) (random) (send) (short) (confirm) DIFS = DCF InterFrame Space (wait before data — longest) SIFS = Short InterFrame Space (wait before ACK — shortest) Backoff = Random timer to prevent simultaneous transmission
1
Sender senses the channel — is it free for DIFS duration?
2
If free: wait a random backoff time, then sense again
3
Still free: transmit DATA frame
4
Wait for ACK from receiver after SIFS
5
ACK received → Success ✅ | No ACK → Retransmit with larger backoff ❌
2. PCF — Point Coordination Function

PCF is optional and works only in Infrastructure mode. The AP acts as Point Coordinator (PC) and polls each station in turn — contention-free access:

PCF POLLING
AP (Point Coordinator) STA1 STA2 STA3 | | | | |── Poll STA1 ──────►| | | |◄── Data ───────────| | | |── Poll STA2 ──────────────►| | |◄── Data ──────────────────| | |── Poll STA3 ─────────────────────►| |◄── Data ─────────────────────────| (Only polled station can transmit — NO collisions!)

Definition of WiMax
WiMax (Worldwide Interoperability for Microwave Access), standardized as IEEE 802.16, is a wireless broadband communication technology that provides high-speed internet access over long distances. Range: up to 50 km, Speed: up to 70 Mbps, Frequency: 2 GHz to 66 GHz. Uses OFDM at physical layer.
WiMax Architecture
WIMAX NETWORK
[INTERNET / Core Network (CSN)] | [Base Station (BS)] ← covers up to 50 km radius / | \ [SS1] [SS2] [MS3] Home Antenna Office Mobile Device SS = Subscriber Station | MS = Mobile Station
Advantages of WiMax
  • Long Range Coverage: Covers up to 50 km in line-of-sight mode, making it ideal for providing broadband in rural and remote areas where laying cables is impractical.
  • High Data Rate: Provides speeds up to 70 Mbps, sufficient for broadband internet, video streaming, and VoIP applications.
  • Last Mile Solution: Solves the last-mile problem by wirelessly connecting end users to the internet backbone without expensive cable infrastructure.
  • Supports Mobility: Mobile WiMax (802.16e) supports handoff between base stations, allowing users to stay connected while moving.
  • Quality of Service (QoS): Supports 5 QoS scheduling types (UGS, rtPS, nrtPS, BE, ertPS) ensuring prioritization of time-sensitive traffic like VoIP.
  • Cost Effective: Cheaper to deploy than laying fiber or cable infrastructure, especially in difficult terrain.
  • NLOS Support: Non-Line-of-Sight operation means signals can travel through obstacles like buildings and trees.
  • Scalable: Network capacity can be easily scaled by adding more base stations to cover larger areas.
WiMax = WiFi on steroids. Fixed WiMax = 802.16d | Mobile WiMax = 802.16e. WiMax MAC is CONNECTION ORIENTED unlike WiFi!

Explain role of SMTP and POP protocol in E-mail message transfer.
7 Marks
▾
✍ Answer
Overview
Email communication requires two different protocols working together: SMTP for sending/forwarding emails between servers, and POP3 for retrieving emails by end users. Think of SMTP as the postal delivery van, and POP3 as collecting your letters from the mailbox.
Role of SMTP — Simple Mail Transfer Protocol (Port 25, TCP)

SMTP is responsible for sending and forwarding emails using a store-and-forward mechanism:

1
Client to Server: When Alice clicks "Send", her email client uses SMTP (Port 25) to push the email to her outgoing mail server.
2
Server to Server: Alice's mail server uses SMTP again to forward the email to Bob's mail server. This is the store-and-forward mechanism — each server stores temporarily before forwarding.
3
Email Stored: The email is stored at Bob's mail server waiting for Bob to retrieve it.
SMTP COMMUNICATION
CLIENT ─────────────────────────────────────────────► SMTP SERVER HELO client.com 220 Service Ready MAIL FROM: alice@a.com 250 OK RCPT TO: bob@b.com 250 OK DATA 354 Start sending [email body] "." 250 Message accepted ✅ QUIT 221 Bye
Role of POP3 — Post Office Protocol v3 (Port 110, TCP)

POP3 is responsible for retrieving emails from the mail server to the user's device:

1
Authentication: Bob's email client connects to his mail server via POP3 (Port 110) and authenticates with USER + PASS commands.
2
Download: Using RETR command, Bob's client downloads emails from server to local device.
3
Delete: Using DELE command, emails are marked for deletion from server. On QUIT, they are permanently deleted.

Definition with Example
DNS (Domain Name System) is a hierarchical, distributed, and decentralized naming system that translates human-readable domain names into IP addresses.

Key Characteristics of DNS
  • Distributed: No single server stores all records — information is spread across millions of servers worldwide for scalability and fault tolerance.
  • Hierarchical: Organized in a tree structure from Root at top, through TLD servers, down to Authoritative servers.
  • Decentralized: No single point of failure — if one server fails, others continue to serve requests.
  • Port 53: Primarily uses UDP for queries (fast); TCP for zone transfers or large responses.

IEEE 802.11 Frame Format Overview
The IEEE 802.11 frame is the basic unit of data transmission in WiFi networks. It consists of three main parts: Header, Body (Payload), and FCS. Notably, it has four address fields unlike Ethernet which has only two.
Complete Frame Structure

1. Frame Control (2 bytes) — Most Important

2. Duration/ID (2 bytes)

Specifies how long the channel will be occupied in microseconds. Other stations read this value to set their NAV (Network Allocation Vector) — a virtual carrier sensing timer. Stations defer transmission until NAV expires.

3. Address Fields (6 bytes each × 4 = 24 bytes)
AddressNamePurpose

Address 3Destination/Source AddressFinal destination or original source MAC

4. Sequence Control (2 bytes)

Contains Sequence Number (12 bits) and Fragment Number (4 bits). Used to reassemble fragmented frames and to discard duplicate frames during retransmission.

5. Frame Body (0 to 2312 bytes)

Contains the actual data payload. Size varies from 0 to 2312 bytes. In management frames it contains beacon info, association requests etc. In data frames it contains actual user data.

6. FCS — Frame Check Sequence (4 bytes)

Contains CRC (Cyclic Redundancy Check) value for error detection. Receiver recalculates CRC and compares with FCS. If they don't match, the frame is discarded and a retransmission is requested.

802.11 has 4 address fields unlike Ethernet (2 addresses). To DS & From DS bits determine the meaning of each address. FCS = error detection via CRC!
Q4b
What are hidden station and exposed station problems in WLAN.
8 Marks
▾
✍ Answer
1. Hidden Station Problem
Definition: A hidden station is a node that is out of range of another transmitting node, but both communicate with the same Access Point. This causes collisions at the AP without either station knowing about it.
HIDDEN STATION DIAGRAM
Range of A Range of B ┌──────────────────┐ ┌──────────────────┐ │ │ │ │ │ [A] ─────────────[AP]───────────── [B] │ │ │ │ │ └──────────────────┘ └──────────────────┘ A cannot hear B ❌ B cannot hear A ❌ Both can hear AP ✅
What Happens Without Solution
1
Station B is already transmitting data to AP
2
Station A senses the channel — hears NOTHING (B is out of range!)
3
Station A assumes channel is free and starts transmitting
4
Both signals arrive at AP simultaneously — COLLISION ❌
5
Neither A nor B knows a collision occurred! AP receives garbled data
Solution — RTS/CTS Mechanism
RTS/CTS SOLUTION
STATION A ACCESS POINT STATION B | | | |── RTS ────────────►| | | "Ready to Send" | | | |── CTS ────────────►| |◄── CTS ────────────| "Channel busy! | | "Go ahead A!" | I must WAIT ⏳" | | | | |── DATA ───────────►| | |◄── ACK ────────────| | | | Now B can send ✅ |

The AP broadcasts the CTS, which is heard by ALL stations including hidden ones. Any station hearing CTS knows the channel is reserved and must wait.

2. Exposed Station Problem
Definition: An exposed station is a node that unnecessarily stops transmitting because it hears a nearby transmission — even though its own transmission would NOT cause any collision at the intended receiver.
EXPOSED STATION DIAGRAM
Range of B Range of C ┌──────────────────────┐ ┌──────────────────────┐ │ │ │ │ │ [A] ◄────── [B] ─────────────── [C] ──────► [D] │ │ │ │ │ └──────────────────────┘ └──────────────────────┘ C is transmitting to D B wants to transmit to A B hears C → B waits unnecessarily! But B→A would NOT interfere with C→D at all!
What Happens
1
Station C is transmitting data to D
2
Station B wants to send data to A (safe — A cannot hear C)
3
Station B senses the channel — hears C transmitting!
4
Station B waits unnecessarily ⏳ — thinking channel is busy
5
B → A transmission would NOT have caused any collision. Bandwidth wasted!
